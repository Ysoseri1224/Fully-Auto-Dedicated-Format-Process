# anti-AI.md —— 文稿质量审查规范

本文件为第九章（及后续章节）撰写与修改的质量标准。每次修改后对照本文自查。

---

## 一、学术谦逊准则

专著的任务是叙述与解释，而非宣传成就，也非自我检讨。两端都须避免：

| 倾向 | 示例 | 问题 |
|------|------|------|
| 过度褒扬 | "发挥着不可替代的作用""代表了最高水平""开创性突破" | 宣传口吻，非学术叙述 |
| 过度谦卑 | "只是方法之一""仅为粗浅尝试""不过是一种思路" | 轻薄化学术工作的分量，同样偏离客观 |

**正确的态度**：客观陈述做了什么、发现了什么、有什么条件和边界。不拔高，不贬低，不评价历史意义。让事实和数据说话。

**审查方法**：搜索全文中的价值判断类词汇。问自己："这句话去掉评价性词汇后，信息量是否减少？"如果回答"否"，则删除该评价。

---

## 二、程度词汇控制

核心原则：**程度词只在陈述可验证的客观事实时使用，不作为修饰性点缀。** 修改时不可矫枉过正导致文风轻薄化——替换词需要与所描述内容的实际分量匹配。

### 禁止级（修饰性夸大，几乎总应删除或降级）

| 禁用词 | 替换原则 |
|--------|---------|
| 根本（的/性）+ 抽象名词（局限/缺陷/原因）| 改为"重要的"或直接陈述事实 |
| 本质区别 | 改为"区别在于" |
| 决定性影响 | 改为"有重要影响"或给出定量描述 |
| 前所未有的 | 删除——真正"前所未有"的工作极其罕见 |
| 标志着 | 删除，直接陈述事实 |
| 最具……（最高级）| 改为"……之一"或删除最高级 |

### 警惕级（根据语境判定保留或修改）

判定方法：如果该词修饰的是一个**可量化或可验证的工程事实**，保留；如果修饰的是一个**评价性的抽象判断**，删除或降级。

| 词 | 可保留的语境（示例） | 应删除的语境（示例） |
|----|-------------------|-------------------|
| 关键 | "界面换热系数是求解温度场的关键输入参数" | "这一发现成为该领域的关键突破" |
| 核心 | "能量守恒是传热方程的物理核心" | "ABAQUS 的核心优势在于……" |
| 挑战 | "糊状区本构参数的高温测量是实验上的一个挑战" | "热裂模拟面临的核心挑战" |
| 极为/极其 | 极少可保留，原则上删除或改为"较" | "极为困难""极其复杂" |
| 大幅/大量 | 给出量化数据时保留（"应变速率增大约3倍"） | 无数据的模糊修饰（"大幅提高"） |
| 系统性 | "该模型系统性地考虑了固相变形和液相流动" | "本文系统性地阐述了……" |
| 理论基石 | 不可使用——没有单个概念能支撑整个理论 | 改为"基础之一"或"的重要组成部分" |

### 特殊处理

- **"根本动力"**：不可用。科学进步的推动因素是多元渐进的。改为"重要推动力"或直接陈述具体机制。
- **"本质"**：用于物理/数学的严格等价时保留（"热传导的本质是声子输运"），用于修辞性强调时删除（"本质区别""本质特征"）。
- **"核心"**：保留原则同上。描述技术系统内不可移除的构成要素时可保留（如在多尺度方法中补缩系数确实是不可或缺的连接参数），但在宣传性语境中（"核心优势""核心工具"）须删除。

## 三、AI 写作痕迹消减

### 3.1 禁用的标点习惯

- ✗ 中文破折号 `——`（AI 高频标记）：替换为逗号、句号、冒号，或拆为两个独立句
- ✗ 连续两个以上 `——` 的嵌套结构

### 3.2 禁用的句式模板

- ✗ "不仅……更……"（AI 偏好句式）
- ✗ "需要强调的是" / "值得注意的是"（注水标记——如果确实值得注意，读者自然会注意到）
- ✗ "发挥着不可替代的作用"及同类评价句式
- ✗ "A 是 B 的 C" 泛化定义句反复出现，尤其当 C 是"核心""基础""关键"等评价词时

### 3.3 允许的模式化表达

以下类型的模式化是**结构性的、服务于可读性**的，不视为 AI 痕迹：

- 技术流程描述句式："A 以 B 为输入，通过 C 求解 D"——这类句式在工程技术文献中正常使用
- 平行结构的列举("一是……二是……"或"(1)……(2)……")——使复杂信息层次清晰
- 节末小结句——如对读者有帮助，保留

### 3.4 疑似编造词汇的自检

- 每个新引入的专业术语需能在本稿引用的参考文献中找到出处
- 描述性的比喻（如"砖块砂浆""桥梁式连接"）可保留，但首次出现时需配合标准术语
- 在参考文献中无法验证的中文译法，改回英文原文或直译

### 3.5 段落组织

- 禁止 bullet point（本文稿未发现）
- `(1) ... (2) ... (3) ...` 项超过 3 行时拆为独立段落
- 避免同一小节内连续使用"首先……其次……最后……"

## 四、逻辑审查清单

修改稿完成后逐一检查：

1. **开头段的承诺与正文的兑现**：声称要讨论 N 个内容，正文是否讨论了 N 个；顺序是否一致
2. **因果链完整性**：A→B→C 的每一步是否有原理或文献支撑，不跳跃
3. **比较的公平性**：比较方法/模型时是否列出了可比的共同条件
4. **术语一致性**：同一概念全文名称统一（首次给出全称+英文，此后用简称）
5. **数值一致性**：同一数据不同位置一致
6. **跨节引用准确性**：引用前文处，被引内容确实存在
7. **读者知识假设一致性**：全文对读者背景知识的假设保持同一水平

## 五、内容适宜性准则

专著面向材料科学与工程领域的研究生和科研人员。

- **基本解释不可回避**：读者具备传热学、凝固原理基础知识，但新引入的专用方法、判据推导思路、模型的核心假设需要给出基本解释。解释的长度以"读者能理解该方法为什么被提出、它的输入输出是什么、适用条件是什么"为准，不延伸到教科书级推导
- **展望的控制**：保留一两句具体方向性提示，删除口号式结语
- **与主题的关联**：每段自问"是否直接服务于理解铝合金热裂数值模拟"
- **跨章内容**：属于前八章的内容（如合金成分影响基础机理）一句带过并标注"详见第 X 章"
